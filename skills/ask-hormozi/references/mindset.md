# Mindset — Converting Feelings into Inputs

Use this guide for fear, focus, motivation, learning, persistence, and decision questions. It is a paraphrased operating guide. Retrieve attribution from `references/evidence/mindset.md`; use `references/evidence/meta.md` for a named decision or learning method.

## Translate the feeling

Convert vague states into observable variables:

| Feeling | Operational question |
|---|---|
| Stuck | Which decision, skill, or repeated action is missing? |
| Unmotivated | Is the next action clear, small, scheduled, and connected to a chosen outcome? |
| Afraid | What specific loss is feared, and is the decision reversible? |
| Overwhelmed | What is the single current constraint? |
| Impatient | What input and time horizon were actually committed? |
| Insecure | What evidence or practice would make competence observable? |

Name the feeling briefly, then move to the controllable input.

## Volume with feedback

Use volume to create evidence, not to avoid thinking. Define a meaningful unit, repeat it long enough to see a pattern, and review the result.

`Skill growth = focused repetitions x feedback quality x time`

If repetitions are low, increase volume. If repetitions are high and results remain flat, inspect the market, method, feedback, or skill rather than prescribing more blindly.

## Focus

Focus is a subtraction decision. Choose one audience, one offer, one primary acquisition path, or one learning target for a defined window. Make the excluded options explicit so they do not remain hidden priorities.

## Decisions

Sort by reversibility:

- **Reversible:** Decide quickly, set a review date, and learn from action.
- **Hard to reverse:** Identify the assumptions that must be true, test the cheapest one first, and protect downside.

For a difficult personal decision, write the problem, then answer it from the perspective of an older future self with aligned incentives and full life context. Treat the result as a perspective tool, not prophecy.

## Learning and identity

- Identify the skill gap before treating the gap as a personality defect.
- Buy or seek instruction when it compresses a costly trial-and-error loop.
- Build confidence from completed repetitions and evidence.
- Treat failure as information about a method or attempt, not a permanent identity.
- Extend the time horizon only when the underlying game remains worth playing.

Persistence is not an excuse to ignore health, ethics, insolvency, or evidence that the chosen vehicle is wrong.

## Completion check

A mindset answer is complete when it converts the stated feeling into one observable constraint, one controllable input, and one action the user can complete today.
