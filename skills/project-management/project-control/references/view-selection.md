# Adaptive view selection

Choose the smallest visual that answers the user's question. The ORBIT showcase views are candidates, not a template.

## View signals

| Relationship the sources establish | Useful view | Required signal |
|---|---|---|
| Explicit blocking or enabling relationships | Dependency map | Named items and sourced edges |
| Dated work across owners or lanes | Execution timeline | Supported dates or ordered periods plus owners |
| Criteria, claims, tests, approvals, or receipts | Evidence ledger | Criterion-to-proof relationships |
| A user-named variable or tradeoff | Scenario lab | Explicit assumptions and transparent rules |
| Unresolved choices shaping downstream work | Decision map | Questions, options, and consequences |
| Stages, systems, or handoffs | Workflow map | Named stages and transitions |
| Release or handoff gates | Readiness view | Gates and their evidence |
| Work distributed across people or teams | Ownership map | Supported owner-to-item assignments |
| Comparable projects or workstreams | Milestone lanes | Comparable states from similarly fresh sources |

Project-specific relationships may earn a different visual. Prefer the native relationship over this list.

## Composition

1. Pick one dominant view for the immediate question.
2. Add a selected-item detail when selection reveals useful context.
3. Add up to three secondary views only when each exposes a different relationship.
4. Use tabs for distinct views and direct labels for the important meaning.
5. Place freshness and uncertainty beside the affected fact.

## Plain-language actions

Useful selected-item actions include:

- **Explain plainly:** what this means and why it matters
- **Show the blocker:** what prevents movement and the evidence for it
- **What changes if this slips?:** a labeled scenario based on known relationships
- **Plan the next move:** up to three actions, distinguishing facts from recommendations

Follow-up prompts carry the project outcome, selected item, current state, evidence, and uncertainty. End with: “Use plain language. Explain any technical terms.”
